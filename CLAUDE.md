# CLAUDE.md — Development Guidelines for SOPAT

**SOPAT** is a full-stack Next.js portfolio and project management platform for a landscape design company, featuring client-facing pages, an admin dashboard, and complex portfolio/project workflows.

---

## 🏗️ Project Architecture

### Tech Stack

- **Frontend**: React 19, Next.js 16.2.4 (App Router), TailwindCSS 4, TypeScript
- **Backend**: Next.js API Routes, Drizzle ORM with PostgreSQL
- **Database**: Neon PostgreSQL with Drizzle migrations
- **Auth**: Iron-session (admin panel), JWT (email validation tokens)
- **File Uploads**: Cloudinary
- **Email**: Nodemailer (SMTP via Brevo)
- **Forms**: React Hook Form + Zod validation
- **State**: Zustand for client state, TanStack React Query for server state
- **Charts/Data Viz**: Recharts
- **PDF Generation**: @react-pdf/renderer (portfolio exports)

## 🔑 Key Features

### 1. **Public Website**
- Homepage with sections (Hero, Services, Projects, Testimonials, About, Contact)
- Services listing and detail pages
- Project showcase with project detail pages
- Company "propos" (about) pages with video, gallery, values
- Blog integration (WordPress via external API)
- Portfolio PDF generation

### 2. **Admin Dashboard** (`/admin`)
- **Projects**: Create, edit, track phases (Étude → Appel d'Offre → Réalisation → Maintenance)
- **Budget Management**: Budget cards, budget predictions with ML
- **Plant/Equipment Lists**: Track plants used in projects
- **Activity Logs**: Track changes to projects
- **Team Management**: Team member CRUD
- **Maintenance (Entretien)**: Schedule and log maintenance visits
- **Realization (Réalisation)**: Track project execution with photo checkpoints and sign-off
- **ISO/Certifications**: Certifications management
- **Settings**: App-wide settings and configuration
- **Documents**: PDF document management
- **Audits**: Audit log viewing

### 3. **Email Workflows**
- Budget validation tokens (one-time links via JWT)
- Email sent to client for budget approval/changes
- Token pages: `/validate/[token]` and `/edit/[token]`

### 4. **File Management**
- Cloudinary integration for image uploads
- Portfolio PDF export with styled components

## 🛠️ Development Workflow

### Setup
```bash
npm install
# Set up environment variables (.env.local based on .env.example)
npm run db:generate    # Generate Drizzle types
npm run db:migrate     # Run pending migrations
npm run db:seed        # Seed initial data
npm run dev            # Start dev server on http://localhost:3000
```

### Key Commands
```bash
npm run build          # Build for production
npm run start          # Start production server
npm run lint           # Run ESLint
npm run test:api       # Test API endpoints
npm run extract:images # Extract images from Cloudinary
npm run migrate        # Run migration helpers
npm run db:generate    # Generate Drizzle schema types
npm run db:migrate     # Apply Drizzle migrations
npm run db:seed        # Run seed script
```

### Database
- **ORM**: Drizzle ORM (strongly typed)
- **Database**: Neon PostgreSQL
- **Migrations**: Auto-generated SQL by Drizzle — check `db/migrations/` before running
- **Seeding**: Run `npm run db:seed` for initial data

## 🔐 Security & CSP

**Content Security Policy** is configured in `next.config.ts`:
- **Admin routes** (`/admin/*`, `/validate/*`, `/edit/*`): Strict CSP
  - No iframes, limited external scripts
  - Cloudinary widget whitelisted for uploads
- **Public pages**: Permissive CSP
  - Allows Google Maps iframe, external fonts, WordPress CDN

**Auth**:
- Admin panel: Iron-session (server-side sessions)
- Email tokens: JWT with expiry (budget validation/edit flows)

**Sensitive Operations**:
- All budget/project changes require authentication
- Email validation tokens have time limits
- Cloudinary uploads are server-validated

## 📦 Environment Variables

Required in Vercel or `.env.local`:
```
DATABASE_URL               # Neon PostgreSQL connection
SESSION_SECRET            # Iron-session encryption (32+ chars)
JWT_SECRET                # JWT token signing (budget emails)
CLOUDINARY_CLOUD_NAME     # Cloudinary account
CLOUDINARY_API_KEY        # Cloudinary API key
CLOUDINARY_API_SECRET     # Cloudinary secret
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME  # Public Cloudinary name
SMTP_HOST                 # Email SMTP (e.g., Brevo)
SMTP_PORT                 # SMTP port (587)
SMTP_USER                 # SMTP username
SMTP_PASS                 # SMTP password
EMAIL_FROM                # From address (noreply@sopat.tn)
NEXT_PUBLIC_APP_URL       # App URL for links in emails
```

## 📋 Common Workflows

### Adding a New Project Page
1. Create route in `src/app/projects/[slug]/page.tsx`
2. Add project data fetch from database
3. Create components in `src/components/projects/`
4. Add project to database schema if needed

### Adding Admin Features
1. Create page in `src/app/admin/(dashboard)/[feature]/` route
2. Create client component with `'use client'` for interactivity
3. Use Zustand stores in `src/store/` for state
4. Create database helpers in `src/lib/db/[feature].ts`
5. Add API route in `src/app/api/[feature]/` if needed

### Email Workflows
1. Generate JWT token with `jwt.ts` helper
2. Build email link: `${NEXT_PUBLIC_APP_URL}/validate/[token]`
3. Send via Nodemailer in API route
4. Token page validates JWT and renders form

### Database Changes
1. Update schema in `db/schema.ts`
2. Run `npm run db:generate` to auto-generate migration
3. Review SQL in `db/migrations/[number]_*.sql`
4. Run `npm run db:migrate` to apply
5. Update `src/lib/db/*.ts` helpers as needed

## 🎨 Code Style & Patterns

### Component Structure
- **Page components**: Async, fetch data at top level
- **Client components**: Use `'use client'` pragma for interactivity
- **Shared components**: Place in `src/components/` organized by feature
- **Forms**: Use React Hook Form + Zod for validation

### Naming
- Components: PascalCase (`ProjectCard.tsx`)
- Utilities: camelCase (`useScrollReveal.ts`)
- Database tables: snake_case (Drizzle convention)

### Imports
- Absolute imports via `@/*` alias (configured in tsconfig.json)
- Database: `@/db` alias points to `db/index.ts`

### Type Safety
- Strict TypeScript (`strict: true`)
- Drizzle generates types automatically
- Zod for runtime validation on forms

## 🐛 Debugging Tips

### Database Issues
- Check connection string in `DATABASE_URL`
- View Drizzle logs with `DEBUG=drizzle:*`
- Use `npm run db:migrate` to see pending migrations

### Auth Issues
- Iron-session: Check `SESSION_SECRET` length (32+ chars)
- JWT: Verify `JWT_SECRET` and token expiry in `jwt.ts`
- Token pages: Check token validity and format

### Cloudinary Issues
- Verify API keys in dashboard
- Check `CLOUDINARY_CLOUD_NAME` matches account
- Review CSP `img-src` directive

### CSP Violations
- Check browser console for CSP errors
- Update `next.config.ts` headers
- Remember: `/admin/*` routes use strict CSP
